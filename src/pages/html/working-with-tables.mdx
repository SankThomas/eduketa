# Working With Tables

Tables are used to create - obviously - tabulated data. There are a number of elements available to use with the `table` element in order to make data presentable, alongside styling it with some attributes and with CSS as well.

The default structure of a table is to create a `table` tag. It is a block element.

**Example:**

```html
<table></table>
```

With combination of other tags available is the `table` element such as `thead`, `tbody`, `tr` and `td`, you can create headings for the table as well as rows, columns and the data in the table.

<Callout type="error">
    Never use tables to layout your website. Only use tables when you want to present structured data (rows and columns). Tables were not designed to layout your entire website from the navigation links to the footer.    
</Callout>

## Simple table example

```html
<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

  <tr>
    <td>Thomas</td>
    <td>25</td>
  </tr>
</table>
```

The code example above creates the following table:

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

  <tr>
    <td>Thomas</td>
    <td>25</td>
  </tr>
</table>

> It doesn't look like a table does it? This is because an attribute, as well as CSS styling need to be applied to it in order for the table to span the entire width.

### Table elements
1. `tr` - Creates a table row
2. `th` - Creates a table heading
3. `td` - Adds table data to the cells.
4. `thead` - Is used to group table headings together.
5. `tbody` - Is used to group items in the body of the table.
6. `tfoot` - Is used to group table footer items.

### Improved Example

```html
<table width="100%" >
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

  <tr>
    <td>Thomas</td>
    <td>25</td>
  </tr>
</table>
```

In the above code snippet, the table width has been set to `100%`. Therefore, it will output the following:

<br />

<table width="100%" >
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

  <tr>
    <td>Thomas</td>
    <td>25</td>
  </tr>
</table>

<br />

> Table headings are centered by default, and you can adjust the width of the table by adjusting the `width` of the table. For example, you can have a table of with `30%` or one of `3000%`.

HTML Tables always have a border by default (even though they might not be visible). To remove these borders, you can add they `style` attribute to the table and add the property `border-collapse: collapse`. If you want the text to align to the left, you can also add `text-align: left` as well. For example:

```html
<table width="100%" style="border-collapse: collapse; text-align: ;eft;">
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

  <tr>
    <td>Thomas</td>
    <td>25</td>
  </tr>
</table>
```

This will output the following:

<br />

<table width="100%" style={{
  borderCollapse: "collapse",
  textAlign: "left"
}}>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

  <tr>
    <td>Thomas</td>
    <td>25</td>
  </tr>
</table>

When adding more data to your table, make sure that the columns correspond to the number of headings and rows so that you don't get a weird-looking table. For example:

```html
<table width="100%">
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

<!--
Adding new items to the table. Notice that for each row,
there are two table cells corresponding to the only two headings
that we have
-->

    <td>25</td>
  </tr>
  
  <tr>
    <td>Luke</td>
    <td>30</td>
  </tr>
  
  <tr>
    <td>Thomas</td>
    <td>38</td>
  </tr>
</table>
```

The above example will output the following:

<br />

<table width="100%" style={{
  borderCollapse: "collapse",
  textAlign: "left"
}}>
  <tr>
    <th>Name</th>
    <th>Age</th>

  <tr>

  </tr>
</table>


## Advanced Tables

Chances are that in your career as a web developer, you will have to build simple tables like the one above, as well as more advanced tables such as a school timetable.

In such a case, we will employ the same techniques when we built the [Simple Table Example](#simple-table-example) above.

Take a look at the example below which is a table that shows the Timetable for my ficitional school:

      <th>Monday</th>
      <th>Tuesday</th>
      <th>Wednesday</th>
      <th>Thursday</th>

## Tables on mobile

If you have been using a mobile device, you must have noticed by now that the tables we have created so far are not responsive: i.e.

What you can do to such tables is add the `style` and then add a  property called `overflow` and set it to `auto` so that when the table width or content is greater than the device width, it will automatically introduce a horizontal scrollbar for the table only - as opposed to a horizontal scrollbar for the entire website. You don't want that to happen because it will massively break your user's experience.

```html
<table style="overflow: auto">
```
